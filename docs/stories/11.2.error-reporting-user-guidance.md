# Story 11.2: Error Reporting and User Guidance

## Status
Draft

## Story

**As a** user encountering an error,
**I want** clear error messages with remediation guidance,
**so that** I can resolve issues without external help.

## Acceptance Criteria

1. Error messages are clear and human-readable
2. Error context explains what was being done
3. Remediation steps are provided for common errors
4. Error messages include relevant resource IDs
5. Errors are logged to file for debugging
6. Error codes can be used for programmatic handling
7. Sensitive information is never included in errors
8. Error display uses Rich formatting
9. Unit tests achieve 80%+ coverage

## Tasks / Subtasks

- [ ] Task 1: Create Error Message Templates (AC: 1, 2)
  - [ ] Define message format
  - [ ] Include operation context
  - [ ] Human-readable language

- [ ] Task 2: Create Remediation Database (AC: 3)
  - [ ] Map errors to remediation steps
  - [ ] Include CLI commands to fix
  - [ ] Link to documentation

- [ ] Task 3: Include Resource Context (AC: 4)
  - [ ] Include resource type
  - [ ] Include resource ID
  - [ ] Include region

- [ ] Task 4: Implement Error Logging (AC: 5)
  - [ ] Log to ~/.geusemaker/logs/
  - [ ] Include full stack trace
  - [ ] Rotate log files

- [ ] Task 5: Define Error Codes (AC: 6)
  - [ ] Unique code per error type
  - [ ] Programmatically parseable
  - [ ] Document all codes

- [ ] Task 6: Sanitize Errors (AC: 7)
  - [ ] Never include credentials
  - [ ] Never include secrets
  - [ ] Mask sensitive values

- [ ] Task 7: Create Rich Error Display (AC: 8)
  - [ ] Color-coded severity
  - [ ] Formatted remediation steps
  - [ ] Copy-pasteable commands

- [ ] Task 8: Create Unit Tests (AC: 9)
  - [ ] Test message formatting
  - [ ] Test remediation lookup
  - [ ] Test sanitization
  - [ ] Achieve 80%+ coverage
